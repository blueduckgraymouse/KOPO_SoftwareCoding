* 클라이언트와 서버
클라이언트와 서버는 두개의 컴퓨터 프로그램 사이의 관계.
클라이언트 : 다른 프로그램에서 서비스를 요청하는 프로그램(컴퓨터), 사용자 중심
서버 : 그 요청에 응답해주는 프로그램(컴퓨터), 데이터 처리, 업무, 안정성등에 초점
을 의미한다.

* 운영체제(OS)
컴퓨터의 하드웨어와 소프트웨어에서 위치하여 사용자로 하여금 컴퓨터를 손쉽게 사용할 수 있게 해주는 프로그램

* GUI와 TUI
GUI : 그래픽 중심으로 사용자가 프로그램을 제어하는 인터페이스 방식
TUI : 텍스트 중심으로 사용자가 프로그램을 제어하는 인터페이스 방식

* 네트워크 용어
IP: IP는 컴퓨터가 네트워크 상에서 서로를 인식하고 통신하기 위하여 사용하는 번호
라우팅: 네트워크 안에서 데이터를 보낼 때 최적의 경로를 선택하는 과정
DNS: 호스트의 도메인 이름을 호스트의 IP주소로 바꾸거나 그 반대 변환을 수행하는 시스템
NAT : 사설 IP를 공인 IP로 변경에 필요한 주소 변환 서비스, 라우터 등의 장비를 사용하여 다수의 시설 IP를 하나의 공인 IP주소로 변환하는 기술
네트워크 IP를 접속 시마다 임의로 배정하는 유동 IP방식
공인IP : 인터넷 사용자의 로컬네트워크를 식별하기 위해 ISP(인터넷 서비스 공급자)가 제공하는 IP주소, 전세계에 유일한 IP주소
할당주체가 ISP이므로 전세계 유일, 내외부 접근 가능
사설IP : 일반 가정이나 회사 등에 할당된 네트워크의 IP주소, 로컬IP, 가상IP라고 한다. IPv4의 주소 부족으로 인해 서브넷팅된 IP이기 때문에 라우터에 의해 로컬 테느워크 상의 PC나 장치에 할당된다.
할당주체가 라우터이므로 그 네트워크 안에서 유일, 외부 접근 불가능
포트포워딩 : 특정한 포트로 들어오는 데이터 패킷을 다른 포트로 바꿔서 다시 전송해주는 작업, 컴퓨터 네트워크 상에서 패킷이 방화벽이나 라우터 같은 네트워크 게이트를 지날 때 IP주소와 포트 번호 결합의 통신 요청을 다른 곳으로 넘겨주는 네트워크 주소 변환의 응용


*
SSH : 네트워크 상의 다른 컴퓨터에 로그인하거나 원격 시스템에서 명령을 실행하고 다른 시스템으로 파일을 복사할 수 있도록 해주는 프로토콜, 22번 포트 사용.
putty : ssh, telnet, tcp를 통한 원격접속에 사용되는 원격제어 프로그램(가상단말기)
포트 : 네트워크 포트는 TCP나 UDP에서 어플리케이션이 상호구분을 위해서 사용하는 번호, 각 프로토콜의 데이터가 통하는 논리적 통로.

*
웹서버(apache) : HTTP를 통해 웹브라우저에서 요청하는 HTML문서나 오브젝트를 전송해주는 서버, 웹 페이지를 클라이언트로 전달하는게 주된 업무, apache는 가장 많이 사용되는 웹서버이다, 정정 컨텐츠만 처리하도록 기능 분배를 해서 서버 부담을 줄인다.
WAS(tomcat) : 웹서버와 웹 컨테이너(서블릿 컨테이너)의 결합으로 다양한 기능을 구현하여 수행할 수 있는 서버, 요청에 맞는 데이터를 DB에서 가져와 요청에 맞는 결과를 제공하며 자원을 효율적으로 사용

2) 추가적인 용어구글링
프로토콜(http,SMB,ftp,ssh) : 기기 간 통신은 교환되는 데이터의 형식에 따라 상호 합의가 요구되고 이런 형식을 정의하는 규칙의 집합을 프로토콜이라고 한다.
IDE : 코딩, 디버그, 컴파일, 배포 등 개발에 관련된 모든 작업을 하나의 프로그램 안에서 처리하는 환경을 제공하는 소프트웨어
디버그 : 개발 단계에서 발생하는 논리적 오류나 비정상적 연산을 찾아 해결하는 과정
컴파일 : 특정 프로그래밍 언어로 쓰여있는 문서를 다른 프로그래밍언어로 번역하는 프로그램
SCSI / ex4 - SCSI는 컴퓨터와 주변 장치를 연결하는 외부 입출력 버스 구조에대한 일련의 규격이다. Ext4는 리눅스에서 주로 사용하는 파일시스템 중 하나이다.
GNU GRUB - GNU 프로젝트의 부트로더로 대부분의 리눅스 배보판에서 부트로더로 사용된다. 부트로더는 운영체제가 시동되기 이전에 미리 실행되면서 커널이 올바르게 시동되기 위해 필요한 모든 작업을 마무리하고 운영체제를 시동키는 프로그램이다.
이더넷 : 주로 근거리 통신망(LAN)에서 주로 사용되는 네트워크 구성 방식.
ipv4 : 패킷 교환 네트워크 상에서 데이터를 교환하기 위한 프로토콜, 주소체계는 총 12자리, 4부분으로 나뉜다. 인터넷 사용자의 증가로 주소공강 고갈로 인해 IPv6가 등장.
IPv6 : 총 128자리
TCP : 인터넷 사엥서 데이터를 메세지의 형태로 보내기 위해 IP와 함께 사용하는 프로토콜
UDP : 사용자 데이터그램 프로토콜, 데이터그램은 독립적인 곤계를 지니는 패킷, 비연결형 프로토콜
